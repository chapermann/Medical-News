# Database Schema — Medical News

## Overview

The database is designed around persistent search engines that accumulate execution history and deduplicated scientific evidence.

---

# 1. users

```sql id="u1"
id SERIAL PRIMARY KEY,
name TEXT,
email TEXT UNIQUE,
created_at TIMESTAMP
```

---

# 2. search_engines

Represents persistent literature monitoring systems.

```sql id="u2"
id SERIAL PRIMARY KEY,
user_id INTEGER REFERENCES users(id),

name TEXT,
description TEXT,

frequency TEXT, -- daily / weekly
start_date DATE,
end_date DATE,

active BOOLEAN DEFAULT true,

last_execution TIMESTAMP,
created_at TIMESTAMP
```

---

# 3. search_terms

Structured representation of query logic.

```sql id="u3"
id SERIAL PRIMARY KEY,
engine_id INTEGER REFERENCES search_engines(id),

term TEXT,
term_type TEXT, -- mesh / decs / keyword / all_fields

boolean_operator TEXT -- AND / OR / NOT
```

---

# 4. search_executions

Tracks each run of a search engine.

```sql id="u4"
id SERIAL PRIMARY KEY,
engine_id INTEGER REFERENCES search_engines(id),

execution_date TIMESTAMP,

total_found INTEGER,
new_found INTEGER,

status TEXT -- success / error
```

---

# 5. articles

Normalized storage of biomedical literature.

```sql id="u5"
id SERIAL PRIMARY KEY,

pmid TEXT UNIQUE,
doi TEXT,

title TEXT,
abstract TEXT,

journal TEXT,
year INTEGER,

authors TEXT
```

---

# 6. article_engine (deduplication map)

Many-to-many relationship between engines and articles.

```sql id="u6"
engine_id INTEGER REFERENCES search_engines(id),
article_id INTEGER REFERENCES articles(id),

first_seen TIMESTAMP,
last_seen TIMESTAMP
```

---

# 7. reports

Weekly or daily synthesis generated by AI.

```sql id="u7"
id SERIAL PRIMARY KEY,
engine_id INTEGER REFERENCES search_engines(id),

period_start DATE,
period_end DATE,

summary TEXT,
ai_model TEXT,

created_at TIMESTAMP
```

---

# 8. design principles

* PMID is the primary deduplication key
* No article is stored twice globally
* Engines are independent
* Articles can belong to multiple engines
* Execution logs are immutable
